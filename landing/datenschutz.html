2. Die Website (playcounter.app)

Hosting & Server-Logfiles

Die Website wird bei Microsoft Azure Static Web Apps (Microsoft Corporation) gehostet. Beim Aufruf der Seite verarbeitet der Hosting-Anbieter technisch notwendige Zugriffsdaten in Server-Logs, insbesondere die IP-Adresse, Datum und Uhrzeit des Zugriffs, die aufgerufene Seite, übertragene Datenmenge sowie Browser- und Betriebssystemangaben. Diese Verarbeitung ist zum sicheren und stabilen Betrieb der Website erforderlich.

Rechtsgrundlage: Art. 6 Abs. 1 lit. f DSGVO (berechtigtes Interesse am sicheren Betrieb der Website).

PlayCounter führt diese Zugriffsdaten nicht mit Nutzerprofilen zusammen und betreibt keine eigene Reichweitenmessung. Die Aufbewahrung technischer Zugriffs- und Sicherheitsprotokolle richtet sich nach der für den Azure-Dienst konfigurierten bzw. von Microsoft vorgegebenen Protokollaufbewahrung; anschließend werden sie gelöscht oder überschrieben, sofern kein Sicherheitsvorfall oder eine gesetzliche Pflicht eine längere Aufbewahrung erfordert.

Schriftarten (Systemschriften)

Die Website verwendet lokal auf deinem Gerät verfügbare Systemschriften. Es werden keine Schriftdateien von Google Fonts oder anderen externen Schriftdiensten geladen; deine IP-Adresse wird zu diesem Zweck nicht an Dritte übertragen.

Cookies, Tracking & Analyse

Die Website setzt keine Cookies, verwendet kein Tracking und keine Analyse-/Statistik-Tools. Es werden keine Profile gebildet.

Externe Links

Die Website verlinkt auf externe Dienste (u. a. GitHub und Discord). Eine Datenübertragung an diese Anbieter findet erst statt, wenn du einen Link aktiv anklickst und die jeweilige Seite besuchst. Dann gelten die Datenschutzbestimmungen des jeweiligen Anbieters.

3. Die Desktop-App

Lokale Verarbeitung

Zur Erkennung gespielter Spiele liest PlayCounter lokal auf deinem Gerät die Namen der laufenden Prozesse aus. Deine Spielzeiten, dein Verlauf, deine Einstellungen sowie eine zufällig erzeugte Installations-Kennung werden lokal auf deinem Gerät gespeichert. Die aktuelle Desktop-App übermittelt keine Spielzeiten oder Sitzungsverläufe an den Server. Es gibt kein Benutzerkonto. Aus Kompatibilitätsgründen noch vorhandene Endpunkte für ältere Live-Statistik-Versionen verwerfen eingehende Anfragen, ohne neue Aktivitätsdaten zu speichern.

Spiele-Abgleich (Matching)

Um ein laufendes Programm einem Spiel zuzuordnen, sendet die App die Namen bzw. Kennungen der laufenden Programme an unsere API (api.playcounter.app, gehostet bei Microsoft Azure in der EU). Unter Windows wird dabei ausschließlich der Dateiname der ausführbaren Datei übertragen (z. B. Hades2.exe), nicht der vollständige Dateipfad. Unter Linux kann zusätzlich der Programmpfad übertragen werden, um das Spiel zu identifizieren.

Die API verarbeitet diese Angaben, um eine Übereinstimmung zurückzugeben. Ein verlässlicher Dateiname-zu-Spiel-Eintrag kann ohne Verknüpfung zu einer Installations-Kennung als Teil der gemeinsamen Matching-Datenbank gespeichert werden. In technischen Anwendungsprotokollen können angefragte Dateinamen vorübergehend zur Fehlerdiagnose erscheinen; unter Windows werden dort keine vollständigen Programmpfade oder Spielzeiten protokolliert. Bei jeder Netzwerkverbindung wird die IP-Adresse technisch an Azure übertragen und kann in dessen Zugriffs- oder Sicherheitsprotokollen verarbeitet werden.

Rechtsgrundlage: Art. 6 Abs. 1 lit. f DSGVO (berechtigtes Interesse an der Kernfunktion der App, Spiele automatisch zu erkennen).

Community-Spiele-Vorschläge

Wenn du ein noch nicht erkanntes Spiel zur Community-Datenbank vorschlägst, werden Exe-Name, Spielname, ausgewähltes Cover und ggf. die zufällige Installations-Kennung an unsere API übermittelt und in der Matching-Datenbank gespeichert. Die Kennung hilft, Einreichungen während der Prüfung auseinanderzuhalten. Abgelehnte Einträge werden entfernt; bestätigte Match-Daten können als Teil der gemeinsamen Datenbank bestehen bleiben. Rechtsgrundlage: Art. 6 Abs. 1 lit. f DSGVO (berechtigtes Interesse an einer überprüfbaren und zuverlässigen Community-Datenbank).

Feedback

Wenn du aktiv Feedback sendest, werden Feedback-Art, Nachrichtentext, App-Version, Betriebssystem und ggf. die zufällige Installations-Kennung an unsere API übermittelt und gespeichert. Die Übermittlung ist freiwillig. Rechtsgrundlage: Art. 6 Abs. 1 lit. f DSGVO (berechtigtes Interesse an der Bearbeitung von Fehlerberichten und der Verbesserung des Dienstes).

Update-Prüfung

Zur Prüfung und Bereitstellung von Updates ruft die App eine Versionsdatei bzw. Installationsdateien von GitHub Releases (GitHub, Inc.) ab. Dabei wird technisch bedingt deine IP-Adresse an GitHub übertragen. Rechtsgrundlage: Art. 6 Abs. 1 lit. f DSGVO (Sicherheit durch aktuelle Software).

Spiele-Cover (IGDB)

Cover-Bilder zu Spielen werden direkt von images.igdb.com (IGDB, betrieben von Twitch Interactive / Amazon) geladen. Beim Laden dieser Bilder wird technisch bedingt deine IP-Adresse an diesen Anbieter übertragen. Rechtsgrundlage: Art. 6 Abs. 1 lit. f DSGVO (Darstellung von Cover-Abbildungen).

Privacy Policy

2. The website (playcounter.app)

Hosting & server logs

The website is hosted on Microsoft Azure Static Web Apps. When you access the site, the hosting provider processes technically necessary access data in server logs (IP address, date/time, requested page, data volume, browser and OS information) to operate the site securely and reliably. Legal basis: Art. 6(1)(f) GDPR (legitimate interest).

PlayCounter does not combine this access data with user profiles and does not operate its own audience measurement. Technical access and security logs are retained according to the retention configured for the Azure service or specified by Microsoft and are then deleted or overwritten, unless a security incident or legal obligation requires longer retention.

Fonts (system fonts)

The website uses system fonts available locally on your device. It does not load font files from Google Fonts or another external font service, so your IP address is not shared with third parties for this purpose.

Cookies, tracking & analytics

The website uses no cookies, no tracking and no analytics. No profiles are created.

External links

The site links to external services (e.g. GitHub and Discord). Data is only transferred to these providers once you actively click a link and visit the respective site, where their own privacy policies apply.

3. The desktop app

Local processing

To detect the games you play, PlayCounter reads the names of running processes locally on your device. Your playtime, history, settings and a randomly generated install identifier are stored locally on your device. The current desktop app does not transmit playtime or session history to the server. There is no user account. Compatibility endpoints retained for older live-statistics versions discard incoming requests without storing new activity data.

Game matching

To map a running program to a game, the app sends the names/identifiers of running programs to our API (api.playcounter.app, hosted on Microsoft Azure in the EU). On Windows only the executable file name is sent (e.g. Hades2.exe), not the full file path. On Linux the program path may additionally be sent to identify the game. Legal basis: Art. 6(1)(f) GDPR.

The API processes these identifiers to return a match. A reliable filename-to-game mapping may be stored as part of the shared matching database without being linked to an install identifier. Requested filenames may temporarily appear in technical application logs for troubleshooting; on Windows, full program paths and playtime are not logged there. Every network connection necessarily transmits the IP address to Azure, where it may be processed in access or security logs.

Community game suggestions

If you suggest an unrecognised game to the community database, the executable name, game name, selected cover and possibly the random install identifier are sent to the API and stored in the matching database. The identifier helps distinguish submissions during review. Rejected entries are removed; confirmed match data may remain as part of the shared database. Legal basis: Art. 6(1)(f) GDPR (legitimate interest in a reviewable and reliable community database).

Feedback

If you actively send feedback, its type, message, app version, operating system and possibly the random install identifier are sent to and stored by the API. Submission is voluntary. Legal basis: Art. 6(1)(f) GDPR (legitimate interest in processing bug reports and improving the service).

Update check

To check for and deliver updates, the app retrieves a version file and installation files from GitHub Releases (GitHub, Inc.). This technically transmits your IP address to GitHub. Legal basis: Art. 6(1)(f) GDPR.

Game covers (IGDB)

Game cover images are loaded directly from images.igdb.com (IGDB, operated by Twitch Interactive / Amazon). Loading these images technically transmits your IP address to this provider. Legal basis: Art. 6(1)(f) GDPR.

4. Contact by email

If you contact us by email, we process your email address, message and usual technical metadata to answer the request. Providing this data is voluntary. The legal basis is Art. 6(1)(f) GDPR and, for contract-related enquiries, also Art. 6(1)(b) GDPR. Google Gmail is used as the email service.

5. Recipients & international transfers

Recipients and services used are Microsoft Azure (website and API hosting), the PostgreSQL hosting provider configured for the API (server-side matching, suggestion and feedback data), GitHub (updates, downloads and source repository), IGDB / Twitch / Amazon (cover images), and Google (email communication). These companies are based in the United States or may process data from there. Transfers to certified US recipients may rely on the EU-US Data Privacy Framework adequacy decision. Where that decision does not cover a recipient or processing operation, the provider's EU Standard Contractual Clauses or another safeguard under Article 46 GDPR is used. Information about the safeguard used for a particular transfer can be requested by email.

6. Retention

  • Local playtime, history and settings remain on your device until you delete them in the app or through your operating system.
  • Matching requests are not stored in the database as a personal gaming history. Non-personal filename-to-game mappings may remain permanently.
  • Technical provider and application logs are deleted or overwritten under the configured retention rules once no longer required for operation, troubleshooting and security.
  • Feedback is stored until processing is complete and afterward only as long as required for traceable product decisions or the defence of possible legal claims.
  • Community suggestions are stored pending review. Confirmed match data may remain while part of the shared database; rejected entries are removed.
  • Emails are deleted after the request is complete unless statutory retention obligations or possible legal claims require further storage.

7. Your rights

Under the GDPR you have the right to:

  • access (Art. 15)
  • rectification (Art. 16)
  • erasure (Art. 17)
  • restriction of processing (Art. 18)
  • data portability (Art. 20)
  • object to processing based on legitimate interest (Art. 21) – e.g. by running the app offline
  • lodge a complaint with a supervisory authority (Art. 77), e.g. the data protection authority of North Rhine-Westphalia (LDI NRW)

To exercise your rights, an informal message to zesarex@gmail.com is sufficient. As we keep no accounts and do not link data sent to the API to your identity, we can only attribute individual records to a user to a limited extent.

8. Voluntary provision and automated decisions

Feedback, community suggestions and contacting us by email are voluntary. PlayCounter can continue to be used locally without an online connection; automatic game matching, cover images and updates are then unavailable. There is no automated decision-making within the meaning of Article 22 GDPR and no profiling.

Note: You can stop the app from transmitting data at any time by running the app offline. The app also works without a server connection; only automatic game matching, cover-image loading and update checks are then unavailable.
